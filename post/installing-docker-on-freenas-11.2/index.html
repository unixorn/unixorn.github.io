Docker on FreeNAS 11#

It isn’t as simple as just clicking on for a docker service in the UI. You’re going to have to set up a VM yourself.

Create Rancher VM#

  1. Create a VM in the new UI, select Docker Host. This will create a Rancher VM that already has docker installed inside so you don’t have to hassle with installing.
  2. Go into the classic webui, select the VM you just created, then devices, and edit
  3. Pick the RAW item, select edit, then set a password. This will be the rancher user’s password.
  4. Back in the new UI, go into VM, select the Rancher VM, hit the … and select serial to see the serial console. At the end of the boot messages, it will print something like docker-sys: 172.18.42.1 eth0: 10.9.8.7 lo: 127.0.0.1 before it presents a login prompt. The username is rancher, and the password is what you set in step 3.

Setup Network on the Rancher VM#

I chose to set the static address for my Rancher VM on my DHCP server. This makes it easier to change later without having to hand-configure the VM, I can just update it on the Unifi controller and reboot the VM.

Hand configuration#

If you want to set up networking inside the VM instead of from the DHCP server, I’m documenting those steps here.

I’m assuming your Rancher showed eth0 as your network interface in the initial setup above, change eth0 in the following steps to whatever yours used.

  1. sudo ros config set rancher.network.interface.eth0.address 10.9.8.7/24
  2. sudo ros config set rancher.network.interface.eth0.gateway YOURGATEWAY
  3. sudo ros config set rancher.network.interface.eth0.mtu 1500
  4. sudo ros config set rancher.network.interface.eth0.dhcp false
  5. sudo ros config set rancher.network.dns "['8.8.8.8','8.8.4.4']"

Create share for docker/rancher to use#

First create a user and group#

  1. Go into Account/Groups, then create a new group. I called mine rancherfiles
  2. Go into Account/Users, create a new user. I called mine rancheruser. Set its primary group to the one you just created, rancherfiles, and uncheck the box to creates a new primary group for it.

Create a dataset#

This is where we’re going to store all the files used by our docker containers.

  1. Open the classic UI.
  2. Go to Storage/Volumes
  3. Select your volume, click Create Dataset
  4. Give the dataset a name, leave the options at their defaults
  5. Select the dataset (you may have to click another chiclet and then click back to make it refresh the list), then click Modify Permissions. Set the owner to rancheruser and the group to rancherfiles, then check the Set Permissions Recursively checkbox and save.

NFS Sharing#

FreeNAS side#

  1. Go to Services, then set NFS to start at boot
  2. Go to Sharing, then create a new UNIX share
    1. Set the maproot user to root
    2. Set the maproot group to wheel
    3. Set the Authorized Hosts and IP addresses to the IP address of your Rancher VM
  3. Save your new share

In an ideal world you’d use maproot user and group to unprivileged accounts, but many docker containers don’t cope well with being unable to massage the permissions of their data directories, so you’re either going to have to trust your containers or spend a lot of time debugging.

This is why we made a separate dataset to share to Rancher.

Rancher side#

  1. Get a root shell with sudo su -
  2. cd /var/lib/rancher/conf/cloud-config.d
  3. Set up NFS - wget https://raw.githubusercontent.com/walkerk1980/docker-nfs-client/master/rancheros-cloud-config.yml
  4. vi rancheros-cloud-config.yml and set SERVER to the IP address of your server, and SHARE to the path to the NFS share you created.
  5. Reboot the rancher VM
  6. You should now be able to see files in /mnt/nfs-1 inside the rancher VM

Install portainer#

Assuming you set up NFS using walkerk1980’s rancheros-cloud-config.yml, you can install portainer with docker run -d -p 9000:9000 -v /var/run/docker.sock:/var/run/docker.sock -v /mnt/dockershare/portainer:/data --restart always --name portainer portainer/portainer

Video to watch https://www.youtube.com/watch?v=RNhqvx8y_8A

single mount#

#cloud-config

# /var/lib/rancher/conf/cloud-config.d/nfs.yml

# https://github.com/rancher/os/issues/641
write_files:
  - path: /etc/rc.local
    permissions: "0755"
    content: |
      #!/bin/bash
      [ ! -e /usr/bin/docker ] && ln -s /usr/bin/docker.dist /usr/bin/docker

rancher:
  services:
    nfs:
      image: walkerk1980/rancher-nfs-client
      labels:
        io.rancher.os.after: console, preload-user-images
        io.rancher.os.scope: system
      net: host
      privileged: true
      restart: always
      volumes:
      - /usr/bin/iptables:/sbin/iptables:ro
      - /mnt/nfs-1:/mnt/nfs-1:shared
      environment:
        SERVER: X.X.X.X
        SHARE: /shared_path

multimount#

#cloud-config
write_files:
  - path: /etc/rc.local
    permissions: "0755"
    owner: root
    content: |
      #!/bin/bash
      mkdir -p /mnt/dockershare
      mkdir -p /mnt/plex
      cloud-init-execute
rancher:
  services:
    nfs:
      image: d3fk/nfs-client
      labels:
        io.rancher.os.after: console, preload-user-images
        io.rancher.os.scope: system
      net: host
      privileged: true
      tty: true
      restart: always
      volumes:
        - /usr/bin/iptables:/sbin/iptables:ro
