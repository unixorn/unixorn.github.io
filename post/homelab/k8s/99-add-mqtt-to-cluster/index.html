Moving mosquitto into the cluster.

Overview#

I run Home Assistant in my homelab and it uses mqtt as a message bus for both zigbee2mqtt and zwave-js-ui. I was running mosquitto in docker on one of the utility VMs in my proxmox cluster. In this post I will document how I moved it into my Talos kubernetes cluster to make it more resilient.

Here’s an overview of how we’re going to set it up to run in k8s.

Namespace: mqtt
 ├─ ConfigMap                    → mosquitto.conf (points to the password file & persistence)
 ├─ Secret                       → The hashed password file (password.txt)
 ├─ PersistentVolumeClaim (PVC)  → Storage for /mosquitto/data - 1Gi is probably overkill
 ├─ StatefulSet                  → Mosquitto container, mounts Secret, ConfigMap, PVC
 └─ Service (LB)                 → External TCP:1883 reachable from anywhere

Configure k8s to run the mosquitto mqtt broker#

Create a mqtt directory to hold the manifest files.

I like to keep my services tidy, so the first thing we’re going to do is create a new mqtt namespace.

# mqtt-namespace.yaml
apiVersion: v1
kind: Namespace
metadata:
  labels:
    kubernetes.io/metadata.name: mqtt
  name: mqtt

Create a ConfigMap#

Use a ConfigMap to create the mosquitto.conf file.

# mqtt-configmap.yaml
# =========================================================
# Create a ConfigMap for the Mosquitto configuration
# =========================================================
apiVersion: v1
kind: ConfigMap
metadata:
  name: mosquitto-config
  namespace: mqtt
data:
  mosquitto.conf: |
    # Listen on the standard MQTT port
    listener 1883

    # Disallow anonymous connections – we only want password auth
    # allow_anonymous false
    allow_anonymous true

    # Path to the password file (mounted from the Secret)
    password_file /mosquitto/config/password.txt

    # Enable persistence and tell Mosquitto where to store its DB
    persistence true
    persistence_location /mosquitto/data/

    log_dest stdout
    log_dest file /mosquitto/log/mosquitto.log

Create a PVC#

If the pod restarts, we don’t want to lose the data for any retained messages, so let’s create a PersistentVolumeClaim to store them.

# mqtt-pvc.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: mosquitto-data-pvc
  namespace: mqtt
spec:
  accessModes:
    - ReadWriteOnce # most block storage supports this
  resources:
    requests:
      # You're going to have to retain a lot of messages to use 1Gi but
      # disk is cheap. Gigabytes are the new megabytes. In the unlikely
      # event that you do need more space you can increase the storage size.
      storage: 1Gi
  # If you have a specific StorageClass, replace “my-storage-class” with its name.
  # Omit the line to use the cluster default.
  # storageClassName: my-storage-class

Create a StatefulSet#

Why a StatefulSet and not just a Deployment?

  • Stable network identity – each pod gets a predictable DNS name (mosquitto-0.mqtt.svc.cluster.local). Useful if you think you will ever need to reference a specific broker instance for monitoring.
  • Stable storage – each replica gets its own PersistentVolumeClaim (PVC) derived from a volumeClaimTemplates block. Even if a pod moves to another node, it will keep the same PVC name, making it easy to script other tooling to back the PVC up

I’m setting the StatefulSet replica count to 1 with replicas: 1 - I’m using mqtt for Home Assistant, and only need a single broker to replicate the old docker setup that has worked well for years. If you decide you need more brokers you can bump the replica count and reapply the manifest and each new pod will automatically get its own PVC. Updating the mosquitto configuration to make it HA is out of scope for this post.

Specify Resource Requirements#

The default Talos configuration will complain if you don’t specify container resource requirements and SecurityContext. It’s a good idea anyway to specify the resources your pod is going to need instead of relying on defaults that could change in a future kubernetes version. It will help the cluster schedule the pods more efficiently.

mosquitto doesn’t need any enhanced capabilities, so we’re going to drop them all.

Here are some reasonable starting points for resources. Monitor with kubectl top pod -n mqtt and adjust as necessary.

  • CPU request: 100m (roughly ≈0.1 core) – this will guarantee that the scheduler reserves a slice.
  • CPU limit: 250m – Kubernetes will let pods burst usage, so set a hard stop for that
  • Memory request 128Mi – This should be enough for the broker and a small in‑memory queue.
  • Memory limit 256Mi – OOM‑kill will kill the pod if it exceeds the memory limit to rotect the node.
# =========================================================
# StatefulSet – the actual broker
# =========================================================
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mosquitto
  namespace: mqtt
  labels:
    app: mosquitto
spec:
  serviceName: mosquitto-headless   # ties the StatefulSet to the headless svc
  replicas: 1                       # increase to >1 if you ever need multiple brokers
  selector:
    matchLabels:
      app: mosquitto
  template:
    metadata:
      labels:
        app: mosquitto
    spec:
      # -------------------- POD‑LEVEL SECURITY --------------------
      securityContext:
        runAsNonRoot: true
        runAsUser: 1883
        runAsGroup: 1883
        fsGroup: 1883
      containers:
      - name: mosquitto
        image: eclipse-mosquitto:2.0   # pin a tag for reproducibility
        ports:
        - containerPort: 1883
          name: mqtt
        # -------------------- RESOURCES --------------------
        resources:
          requests:
            cpu: "100m"
            memory: "128Mi"
          limits:
            cpu: "250m"
            memory: "256Mi"
        # -------------------- CONTAINER SECURITY --------------------
        securityContext:
          readOnlyRootFilesystem: true
          allowPrivilegeEscalation: false
          capabilities:
            drop: ["ALL"]
          seccompProfile:
            type: RuntimeDefault
        # -------------------- VOLUME MOUNTS --------------------
        volumeMounts:
        # ConfigMap → /mosquitto/config (read‑only)
        - name: mosquitto-conf
          mountPath: /mosquitto/config
          readOnly: true
        # Secret → password.txt (single file, read‑only)
        - name: mosquitto-passwd
          mountPath: /mosquitto/config/password.txt
          subPath: password.txt
          readOnly: true
        # PVC → persistent data directory (read‑write)
        - name: mosquitto-data
          mountPath: /mosquitto/data
  # -------------------- VOLUME CLAIM TEMPLATE --------------------
  volumeClaimTemplates:
  - metadata:
      name: mosquitto-data          # each pod gets a PVC named <statefulset>-<claim>-<ordinal>
    spec:
      accessModes:
      - ReadWriteOnce
      resources:
        requests:
          storage: 5Gi              # adjust upward if you expect a lot of retained msgs
      storageClassName: standard   # replace with your cluster’s default or a specific class
  # -------------------- VOLUMES (ConfigMap & Secret) --------------------
  volumeMounts: []   # placeholder – actual mounts are defined in the container spec
  volumes:
  - name: mosquitto-conf
    configMap:
      name: mosquitto-config
      items:
      - key: mosquitto.conf
        path: mosquitto.conf
  - name: mosquitto-passwd
    secret:
      secretName: mosquitto-passwd
      items:
      - key: password.txt
        path: password.txt

Services#

MQTT won’t be useful if devices and dependent services running outside the cluster can’t reach it, so we need to set up services - one headless one that causes us to get the nice consistent DNS name (mosquitto-0.mqtt.svc.cluster.local) and one that does the actual work of routing external traffic to the pod created by the StatefulSet.

# mqtt-services.yaml
# =========================================================
# StatefulSet DNS requires an headless service so that it
# gets a consistent DNS name inside the cluster
# =========================================================
apiVersion: v1
kind: Service
metadata:
  name: mosquitto-headless
  namespace: mqtt
  labels:
    app: mosquitto
spec:
  clusterIP: None  # makes it headless
  selector:
    app: mosquitto
  ports:
  - protocol: TCP
    port: 1883
    targetPort: 1883
    name: mqtt
